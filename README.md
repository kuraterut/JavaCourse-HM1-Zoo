Сперва настроим входной и выходной поток в Main(ввод и вывод с консоли
или воспользуемся файлами)
Далее:

Запуск производится с помощью maven командой:

```mvn clean exec:java -Dexec.mainClass="org.kuraterut.Main"```

Для добавление новой сущности необходимо лишь создать ее и унаследовать соответствующие 
абстрактные классы/интерфейсы (Herbo, Predator, Employee, Thing...), И добавить его в map в ```Generator```

Интерфейс приложения - три команды:
1) ```add *entity*``` - добавление соответствующей сущности, далее последуют вопросы для
информации о сущности(например для мартышки - имя, кол-во еды в день, здоровье и доброта)

2) ```info``` - вывод информации о зоопарке:

2.1) ```info``` - без аргументов выводит всю информацию о зоопарке

2.2) ```info animals``` - выводит информацию о всех животных

2.3) ```info contact``` - выводит информацию о всех животных в контактном зоопарке

2.4) ```info inventory``` - выводит информацию обо всем инвентаре

2.5) ```info employees``` - выводит информацию о всех сотрудниках

2.6) ```info food``` - выводит информацию о всех сотрудниках


3) ```end``` - завершение работы

Принципы SOLID были соблюдены:

1) Все классы отвечают лишь а что-то одно.

2) Open-Close принцип соблюден - Для добавления новых классов не нужно менять существующий код

3) Принцип подстановки Лискова - Подклассы не изменяют поведения базовых классов

4) Интерфейсы определены максимально корректно

5) Были внедрены DI-контейнеры в виде Generator'а, который создает объект с необходимыми параметрами
